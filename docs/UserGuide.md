# BlenderRigConvert 사용 가이드

## 사전 요구사항

- Blender **4.5 LTS**
- **Auto-Rig Pro** 애드온 설치 및 활성화 (유료 애드온)
- BlenderRigConvert 애드온 설치 및 활성화

---

## 설치

1. Blender → Edit → Preferences → Add-ons → Install
2. `arp_convert_addon.py`가 포함된 `scripts/` 폴더를 zip으로 묶어 설치하거나,
   팀에서 공유된 설치 파일을 사용한다
3. 설치 후 "ARP Convert" 애드온이 활성화되어 있는지 확인한다
4. Blender 3D 뷰포트 → N 패널 → **ARP Convert** 탭이 보이면 준비 완료

---

## 전체 워크플로우 요약

```
기존 리그 준비
      ↓
  1. 분석      — 본 구조 자동 분석 + 프리뷰 생성
      ↓
  2. 역할 수정  — 틀린 역할만 수동으로 수정
      ↓
  3. 리그 생성  — ARP 리그 자동 생성
      ↓
  4. 리타겟    — 기존 애니메이션을 ARP 리그에 베이크
      ↓
  5. 정리      — 소스/프리뷰 삭제, 액션 이름 정리
```

---

## 1단계: 분석

**목표**: 기존 리그의 본 구조를 읽고 역할을 자동 추론한다.

1. 변환할 아마추어를 선택한다
2. N 패널 → ARP Convert → **1. 분석** 패널 열기
3. **소스 아마추어** 필드에 대상 아마추어가 표시되는지 확인
4. **Create Preview** 버튼 클릭

완료되면:
- 뷰포트에 프리뷰 아마추어가 생성된다 (원본 아마추어와 겹쳐서 표시됨)
- 패널 헤더에 **신뢰도 %** 가 표시된다 (예: "신뢰도 87%")
- 신뢰도가 낮을수록 다음 단계에서 수동 수정이 많이 필요하다

> **프리뷰 아마추어란?**
> 원본 리그를 건드리지 않고 역할 정보를 시각화하기 위한 임시 아마추어다.
> 본 색상으로 역할을 표시한다 (파랑 = 척추, 빨강 = 뒷다리, 초록 = 앞다리 등).

---

## 2단계: 역할 수정

**목표**: 자동 추론이 틀린 본의 역할을 수정한다.

1. N 패널 → **2. 역할 수정** 패널 열기
2. **계층 트리**에서 본 이름을 클릭하면 뷰포트에서 해당 본이 선택된다
   - 트리의 연결선(`├─`, `└─`)으로 부모-자식 관계를 파악할 수 있다
   - 본 이름 옆에 현재 역할이 한국어로 표시된다
3. 역할이 틀린 본을 선택한 뒤, 아래 **역할 버튼**에서 올바른 역할을 클릭한다

### 역할 목록

| 역할 | 설명 |
|------|------|
| 루트 | 전체 아마추어의 최상위 본 |
| 궤적 | 루트의 부모 본 (있는 경우) |
| 스파인 | 등뼈 체인 |
| 목 | 목 체인 |
| 머리 | 머리 본 |
| 뒷다리 L/R | 뒷다리 상부 (2~3본) |
| 뒷발 L/R | 뒷발 + 발가락 |
| 앞다리 L/R | 앞다리 상부 (2~3본) |
| 앞발 L/R | 앞발 + 발가락 |
| 귀 L/R | 귀 체인 |
| 꼬리 | 꼬리 체인 |
| 미매핑 | ARP에 직접 대응하지 않는 본 (눈, 턱, 혀 등) — cc_ 커스텀 본으로 자동 처리됨 |

### 자주 수정해야 하는 경우

- **플랫(flat) 계층 구조**: 다리 본이 순차로 연결되지 않고 모두 같은 부모 아래 나열된 경우.
  자동 추론이 역할을 "미매핑"으로 잡을 수 있다.
  → 해당 본을 직접 선택하고 올바른 역할(뒷다리 L 등)을 클릭해 수정한다.

- **다리 역할 미매핑**: 구조가 복잡하거나 본 이름이 비표준인 경우.
  → 트리에서 다리 본을 찾아 역할을 수동 지정한다.

> 수정 후 패널 헤더의 "N/M 매핑됨" 숫자가 올라가는 것을 확인한다.

---

## 3단계: 리그 생성

**목표**: 역할 정보를 바탕으로 ARP 리그를 자동으로 생성한다.

1. N 패널 → **3. 리그 생성** 패널 열기
2. (선택) **앞다리 3본 IK** 슬라이더 조정
   - `0`: 어깨가 발 IK와 독립적으로 움직임
   - `1`: 어깨가 발 IK에 완전히 연동됨
   - 대부분의 경우 기본값 그대로 사용한다
3. **Build Rig** 버튼 클릭

완료되면:
- 씬에 ARP 아마추어가 생성된다
- 웨이트가 원본 메시에서 ARP 리그로 자동 전송된다
- 패널 헤더에 체크마크(✓)가 표시된다

> **폴 벡터 위치에 대해**: 다리 IK의 폴 벡터(무릎 방향 가이드) 위치가 이상하게 보일 수 있다.
> 이는 ARP의 기본 동작이며 이 애드온에서 수정할 수 없다.
> 폴 벡터 위치를 조정하려면 ARP 애드온에서 직접 리그를 수정해야 한다.

---

## 4단계: 리타겟

**목표**: 기존 아마추어에 있는 애니메이션을 ARP 리그에 베이크한다.

1. N 패널 → **4. 리타겟** 패널 열기
2. **Setup Retarget** 버튼 클릭
   - 소스 본과 ARP 타겟 본의 매핑 목록이 자동으로 구성된다
   - 목록에서 매핑이 올바른지 확인한다
3. **Re-Retarget** 버튼 클릭
   - 원본 애니메이션을 ARP 리그에 베이크한다
   - 여러 액션이 있는 경우 **다중 소스 애니메이션** 체크박스를 활성화한다
4. **Copy Custom Scale** 버튼 클릭 ← **반드시 실행할 것**
   - ARP 리타겟이 처리하지 못하는 커스텀 본의 스케일 애니메이션을 별도로 복사한다
   - 이 단계를 건너뛰면 커스텀 본(눈, 귀, 꼬리 등)의 스케일 애니메이션이 적용되지 않는다

> **Copy Custom Scale을 빠뜨리면**: 커스텀 본의 스케일 fcurve가 누락된다.
> 애니메이션 재생 시 해당 본이 의도와 다르게 움직이거나 크기가 변하지 않는다.
> Re-Retarget 후 반드시 실행한다.

---

## 5단계: 정리

**목표**: 작업에 사용된 임시 오브젝트와 액션을 정리한다.

1. N 패널 → **5. 정리** 패널 열기
2. **Cleanup** 버튼 클릭
   - 소스 아마추어와 프리뷰 아마추어를 씬에서 제거한다
   - `_remap` 접미사가 붙은 액션 이름을 정리한다

완료 후 씬에는 ARP 리그와 메시만 남는다.

---

## Unity 익스포트

Cleanup 완료 후 ARP의 내장 익스포터를 사용한다.

ARP N 패널 → **Export** 탭 → 대상 엔진으로 **Unity** 선택 → Export 실행.

ARP 익스포터는 컨트롤러 본을 자동으로 제외하고 deform 본만 포함한 FBX를 생성한다.
모든 캐릭터가 동일한 ARP 구조이므로 익스포트 설정을 바꿀 필요가 없다.

## 트러블슈팅 요약

| 증상 | 원인 | 해결 |
|------|------|------|
| 다리 본이 미매핑으로 표시됨 | 플랫 계층 구조 or 비표준 본 이름 | 2단계에서 수동으로 역할 지정 |
| 커스텀 본 애니메이션이 이상함 | Copy Custom Scale 미실행 | Re-Retarget 후 Copy Custom Scale 실행 |
| 폴 벡터(무릎 방향) 위치가 이상함 | ARP 기본 동작 | ARP 애드온에서 직접 리그 수정 필요 |
| 신뢰도가 낮음 (50% 미만) | 본 구조가 비표준 | 2단계에서 역할 전체를 검토 후 수정 |
