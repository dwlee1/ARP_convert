# BlenderRigConvert 제작 배경

## 왜 이 애드온을 만들었는가

### 기존 방식의 문제

동물 캐릭터 리깅 작업은 처음부터 Blender의 기본 리깅으로 진행했다.
작업이 쌓이면서 세 가지 문제가 반복됐다.

---

**1. 통일된 구조가 없었다**

제작 시마다 리깅 방식이 달랐다. 어떤 캐릭터는 본 이름이 다르고, 어떤 캐릭터는 컨트롤러 구조가 다르다.
동물 캐릭터는 기존 리그를 재활용하는 경우가 많은데, 재활용할 때마다 원본 리그의 방식을 그대로 따라가게 된다.
결과적으로 프로젝트 전체에 걸쳐 리깅 방식이 혼재된 상태가 됐다.

**2. Unity 익스포트가 매번 달랐다**

Blender 기본 리그에는 Unity에서 불필요한 컨트롤러 본이 포함된다.
이를 정리하기 위해 게임용 리그 변환 애드온을 추가로 사용했다.
이렇게 되면 **원본 리그**와 **변환된 게임용 리그**가 별도 파일로 분리된다.
캐릭터마다 두 개의 파일을 관리해야 하고, 수정이 생기면 양쪽을 동기화해야 한다.

**3. 익스포트 실수가 반복됐다**

리깅 방식이 캐릭터마다 달랐기 때문에, Unity 익스포트 설정도 캐릭터마다 달라야 했다.
어떤 캐릭터에 어떤 익스포트 설정을 쓰는지 기억해야 했고, 실수가 나오는 경우가 있었다.

---

### Auto-Rig Pro로의 전환

세 가지 문제를 한 번에 해결하는 방법은 **모든 리그를 하나의 표준으로 통일**하는 것이었다.
Auto-Rig Pro(이하 ARP)는 게임 엔진 익스포트 파이프라인이 내장되어 있고,
`dog` 프리셋은 사족보행 동물 구조를 충실하게 지원한다.

ARP로 통일하면:
- 모든 캐릭터의 컨트롤러 구조가 동일해진다
- ARP의 내장 익스포터 하나로 Unity 익스포트가 통일된다
- 파일이 두 개로 분리되지 않는다

---

### 이 애드온이 하는 일

ARP 전환 자체는 수작업이 많다. 기존 리그에서 각 본의 역할(척추인지 다리인지 귀인지)을 파악하고,
ARP ref 본 위치를 일일이 맞추는 작업이 반복된다.

이 애드온은 그 과정을 자동화한다.

1. **자동 역할 추론** — 기존 리그의 본 구조를 분석해 척추·다리·발·귀·꼬리 등의 역할을 자동으로 판별한다
2. **시각적 검토** — 추론 결과를 프리뷰 아마추어로 보여준다. 틀린 부분만 수정하면 된다
3. **ARP 리그 자동 생성** — 역할 정보를 바탕으로 ARP ref 본 위치를 자동으로 정렬하고 리그를 완성한다
4. **리타겟 자동 구성** — 기존 애니메이션을 ARP 리그에 베이크하는 설정을 자동으로 구성한다

수작업으로 반나절 걸리던 변환 작업을 버튼 몇 번으로 완료하는 것이 목표다.
