# 동물 캐릭터 리깅 통일 프로젝트 - 요약

> 작성일: 2026-03-18
> 프로젝트: BlenderRigConvert

---

## 프로젝트 한 줄 요약

**동물 캐릭터 파일의 뼈대(리그) 시스템을 하나로 통일하는 자동화 프로젝트**

---

## 배경

### 현재 상황

3D 동물 캐릭터를 만들 때, 캐릭터가 움직일 수 있도록 내부에 "뼈대(리그)"를 넣습니다.
현재 프로젝트에는 **두 가지 다른 뼈대 시스템**이 섞여 있습니다:

| 시스템 | 비율 | 설명 |
|--------|------|------|
| **Rigify** | ~50% | Blender 내장 도구로 만든 뼈대 |
| **AutoRig Pro (ARP)** | ~50% | 전문 애드온으로 만든 뼈대 |

### 왜 문제인가?

- 작업자마다 다른 방식으로 작업해야 함 → **협업 비효율**
- 게임 엔진으로 내보내기 시 각각 다른 설정이 필요 → **추가 작업**
- 유지보수 복잡성 증가 → **장기적 비용**

### 해결 방향

**ARP 하나로 통일** → 작업 흐름 단순화, 내보내기 품질 일관성 확보

---

## 뭘 하는 건가?

Rigify로 만든 **~500개 파일**을 자동화 스크립트로 ARP 방식으로 변환합니다.

### 변환 과정 (비유)

| 단계 | 비유 | 실제 작업 |
|------|------|-----------|
| 1단계 | 기존 골격 분석 | 현재 뼈대의 위치와 구조를 읽어옴 |
| 2단계 | 새 골격 생성 | 같은 위치에 새로운 ARP 뼈대를 만듦 |
| 3단계 | 동작 복사 | 걷기, 뛰기 등 기존 애니메이션을 새 뼈대로 옮김 |
| 4단계 | 피부 재연결 | 3D 모델(외형)을 새 뼈대에 연결 |
| 5단계 | 검증 | 애니메이션 정상 재생 + 게임용 파일 내보내기 성공 확인 |

이 과정을 **자동화 프로그램**으로 일괄 처리합니다.

---

## 대상 범위

### 포함

- **4족보행 동물**: 고양이, 여우, 곰, 늑대 등
- **조류**: 독수리, 펠리컨, 알바트로스 등

### 제외

- 해양 동물 (돌고래, 범고래 등)
- 기타 특수 동물

---

## 진행 단계

```
Phase 0: 도구 환경 조사
  └→ 자동화에 필요한 기능이 모두 동작하는지 확인
  └→ 기존 오픈소스 도구 활용 가능성 검증

Phase 1: 시범 변환
  └→ 사족보행 동물 1마리로 전체 과정 테스트
  └→ 리그 변환 + 애니메이션 전송 + 게임 파일 내보내기

Phase 2: 문제 해결
  └→ 시범 변환에서 발견된 문제 수정
  └→ 동물마다 다른 뼈대 구조에 대한 대응 로직 개발

Phase 3: 조류 확장
  └→ 조류용 변환 로직 추가
  └→ 날개 등 특수 뼈대에 대한 처리

Phase 4: 전체 일괄 변환
  └→ 모든 대상 파일 자동 변환
  └→ 결과 리포트 생성 (성공/실패/수동 작업 필요 목록)
```

---

## 성공 기준

| 기준 | 목표 |
|------|------|
| 애니메이션 재생 | 변환 후 모든 동작이 정상 재생 |
| 게임 파일 내보내기 | FBX 형식으로 오류 없이 내보내기 성공 |
| 자동화 성공률 | 전체 대상 파일의 **90% 이상** 자동 변환 |

---

## 활용 도구

| 도구 | 용도 |
|------|------|
| Blender 4.5 LTS | 3D 작업 환경 |
| AutoRig Pro (ARP) | 변환 대상 리깅 시스템 |
| ARP Remap | 애니메이션을 다른 뼈대로 옮기는 기능 |
| Python 스크립트 | 자동화 프로그램 |
| ARP-Batch-Retargeting (오픈소스) | 배치 처리 참조 코드 |

## 리스크 & 대응

| 리스크 | 영향 | 대응 방안 |
|--------|------|-----------|
| 자동화 도구가 예상대로 동작하지 않음 | 높음 | Phase 0에서 사전 검증 |
| 동물마다 뼈대 구조가 달라 일부 수동 작업 필요 | 중간 | 카테고리별 설정 분리 (사족보행/조류) |
| 애니메이션 전송 시 품질 저하 | 중간 | 변환 후 검수 단계 포함 |
| 커스텀 추가 본(꼬리, 귀 등) 처리 | 중간 | 매핑 불가한 본은 별도 보존 처리 |

## 기대 효과

- **작업 효율 향상**: 리깅 방식 통일로 모든 작업자가 동일한 워크플로우 사용
- **내보내기 일관성**: FBX 익스포트 설정 통일
- **유지보수 간소화**: 하나의 리깅 시스템만 관리
- **자동화 기반 구축**: 향후 추가 동물에도 동일한 파이프라인 적용 가능
